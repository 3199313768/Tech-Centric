## 1. 文档目的

UI 创意优化（Phase 1–3）已在视觉叙事与场景化上完成一轮升级。本文从**产品设计与信息架构**角度，梳理：

1. 现有模块仍可优化的方向（不限于视觉）
2. 可新增的功能模块与页面
3. 优先级与分阶段路线

**前提**：SpiritGarden「数字庭院」世界观已确立，后续优化应**强化叙事连贯性**，而非推翻现有架构。

## 2. 产品现状诊断

### 2.1 站点地图

**当前**：

```
庭院（/）
├── 归档（/projects）      — 项目列表 + CRUD
├── 技能工坊（/skills）    — 技能列表 + CRUD
├── 草本集（/vibe）        — 笔记/灵感 + CRUD
├── 资源（/resources）     — 外链资源架 + AI 辅助
├── 档案馆（/knowledge）   — 私有知识库（需登录）
└── 庭院导引（FloatingAssistant）— 全站 RAG + Contact
```

**目标（§12 决策后，Q5 修订）**：

```
庭院（/）                 — 顶栏 Tab + Logo 回首页
├── 归档（/projects）      — 列表 + CRUD + /projects/[slug] 详情
├── 技能工坊（/skills）
├── 草本集（/vibe）        — 短笔记 + 长文（合并原博客，不独立 /writing）
├── 资源（/resources）
├── 档案馆（/knowledge）   — CRUD + 记录级 is_public
├── 园主（/about）         — 顶栏 Tab
└── 庭院导引               — 全站 RAG + Contact
```

### 2.2 核心问题（产品层）

| 维度 | 现状 | 问题 |
|------|------|------|
| **身份叙事** | `personal.ts` 含完整履历、技能、博客种子 | 无独立「关于我」页；首页 Hero 文案偏泛，**专业背书未结构化呈现** |
| **内容深度** | 项目/技能/笔记均为**列表页**，无详情路由 | 访客无法深入了解单个作品；SEO 与分享链接缺失 |
| **内容孤岛** | 博客数据在 `personal.ts`，链接为 `#` | 「写作/输出」能力名存实亡，与庭院世界观脱节 |
| **助手价值** | RAG 已全站挂载 | 知识库因 RLS **无法被索引**（见 project-status §3.1），助手回答范围受限 |
| **转化路径** | Contact 仅在助手内 | 无显式 CTA 区块；招聘方/合作方**找不到明确的「下一步」** |
| **数据真实性** | 社交链接仍为 `yourusername` 占位 | 信任感与外链导流均未完成 |
| **所有者体验** | 知识库 Cmd+K 录入体验好 | 子页 CRUD 分散，**无统一「内容工作台」**感知 |
| **可发现性** | 6 个顶栏 Tab + 助手 | Tab 较多，**首次访客认知负荷偏高**；缺少站点级搜索 |

### 2.3 用户画像与核心诉求

| 画像 | 典型场景 | 当前满足度 | 主要缺口 |
|------|----------|------------|----------|
| **招聘方 / HR** | 3 分钟内判断候选人水平 | 中 | 缺结构化简历页、项目详情、明确联系方式 |
| **技术同行** | 看作品、读笔记、借鉴资源 | 中 | 项目无深度页；博客未落地；知识库不可见 |
| **合作方** | 了解能力边界、发起合作 | 低 | Contact 入口隐蔽；无案例研究/交付说明 |
| **站点所有者（你）** | 维护内容、沉淀知识 | 高 | 录入体验好；但公开/私有内容边界模糊 |

## 3. 现有模块优化建议

### 3.1 首页（庭院）

**已做**：Featured 接 DB、动态度量仪、季节 Hero、灵尘粒子。

**仍可优化**：

| 项 | 描述 | 优先级 | 理由 |
|----|------|--------|------|
| **身份一句话** | Hero 副标题接入 `personalInfo.title` + 一句差异化定位（如「水利孪生 · 前端 · 效能」） | P0 | 首屏 5 秒内需回答「你是谁、做什么」 |
| **社会证明条** | Featured 下方增加「13 项目准时交付 / 月度之星 / B+」等可验证指标（来自 `workExperience`） | P1 | 招聘场景决策依据 |
| **CTA 双路径** | 「查看作品」→ `/projects`；「了解我」→ `/about`（顶栏「园主」） | P0 | 降低「只看不转化」 |
| **最新动态** | 卡片展示最近 3 条：最新项目 / vibe 笔记 / 知识库公开摘要（需公开策略） | P2 | 传达「站点是活的」 |
| **Footer 信息架构** | 页脚：**明文邮箱**、GitHub、备案（如有）、©、主题说明 | **P0** | Q3A：邮箱公开展示；全站缺 Footer |

### 3.2 归档（/projects）

| 项 | 描述 | 优先级 |
|----|------|--------|
| **项目详情页** | `/projects/[slug]`：封面轮播、技术栈、职责、链接、相关技能；**数据源：扩展 Supabase `all_projects` 表**（Q4A） | **P0** |
| **筛选与排序** | 按 tag / 年份 / 类型（个人/工作）筛选 | P1 |
| **置顶与精选** | `is_featured` 字段驱动首页 Featured | P1（Featured 已有，需管理 UI） |
| **案例叙事模板** | 结构化字段：背景 → 挑战 → 方案 → 结果（量化） | P2 |
| **分享 OG** | 详情页独立 `metadata` + OG 图 | P2 |

### 3.3 技能工坊（/skills）

| 项 | 描述 | 优先级 |
|----|------|--------|
| **技能 ↔ 项目关联** | 点击技能展示关联项目（`skillsDetail.projects` 已有种子） | P1 |
| **分类视图** | frontend / backend / tools Tab 或分组 | P2 |
| **雷达图 / 星座图** | 视觉化概览（CSS/SVG，非重 JS） | P3 |
| **与 RAG 对齐** | 助手回答技能问题时链到 `/skills#react` | P2 |

### 3.4 草本集（/vibe）

> **Q1B**：博客与 vibe **合并为同一模块**，不独立 `/writing`。短笔记与长文均在 `/vibe` 内，通过类型或篇幅区分（如 `kind: note | article`）。

| 项 | 描述 | 优先级 |
|----|------|--------|
| **长文支持** | 扩展 vibe 表/UI：支持 Markdown 长文；迁移 `blogPosts` 种子 | **P1** |
| **时间线视图** | 除列表外，按月的「生长日志」时间轴（含长文） | P2 |
| **公开精选** | 标记 `is_public` 的笔记/长文可在首页/助手引用 | P1 |
| **详情路由** | `/vibe/[slug]` 或 id 详情页，支撑分享与 RAG 深链 | P1 |

### 3.5 资源（/resources）

| 项 | 描述 | 优先级 |
|----|------|--------|
| **使用场景标签** | 「面试准备 / 性能优化 / 设计灵感」 | P2 |
| **个人评分/备注** | 为何推荐（一句话），增强策展感 | P2 |
| **失效检测** | 定期 HEAD 请求或用户上报 broken link | P3 |
| **导出** | OPML / JSON 导出我的资源架 | P3 |

### 3.6 档案馆（/knowledge）

| 项 | 描述 | 优先级 |
|----|------|--------|
| **公开/私有策略** | **Q2A**：记录级 `is_public`；公开项进 RAG + 公开展柜只读视图 | **P0**（阻塞助手价值） |
| **详情页** | `/knowledge/[id]` 只读分享（公开项） | P1 |
| **全局搜索** | Cmd+K 扩展为「搜索 + 录入」双模式 | P2 |
| **关联图谱** | 同 tag 记录推荐 | P3 |

### 3.7 庭院导引（RAG 助手）

| 项 | 描述 | 优先级 |
|----|------|--------|
| **情境化欢迎** | 按当前路由变换欢迎语与建议问题（如在 `/projects` 推「介绍最新项目」） | P1 |
| **来源可点击** | 回答中的来源链到对应详情页 | P1 |
| **快捷动作** | 「发邮件」「复制邮箱」「跳转 GitHub」 | P1 |
| **Contact 闭环** | 生成邮件草稿后提供「打开邮件客户端」；长期可接 Webhook | P2 |
| **对话历史** | 会话级 localStorage（非账号） | P3 |

### 3.8 导航与 IA

> **Q5 修订**：顶栏保留 **「庭院」** Tab → `/`；Logo 同样回首页；**「园主」** Tab → `/about`。

| 项 | 描述 | 优先级 |
|----|------|--------|
| **顶栏结构** | `SITE_NAV_TABS`：庭院 / 归档 / 技能工坊 / 草本集 / 资源 / 档案馆 / 园主 | **P0** |
| **关于入口** | 顶栏「园主」+ Footer 邮箱 + Hero CTA「了解我」 | P0 |
| **Tab 分组** | 6 Tab 仍偏多；后续可考虑视觉分组（展示 vs 记录） | P2 |
| **面包屑** | 详情页与子页层级导航 | P2 |
| **站点搜索** | 跨项目/技能/vibe/公开知识的统一搜索（可先客户端，后 Server） | P2 |

## 4. 建议新增模块

### 4.1 关于庭园（/about）— **强烈推荐 P0**

**定位**：结构化个人品牌页，补足「我是谁」叙事。

**内容区块**：

```
┌─────────────────────────────────────────┐
│ Hero：姓名 + 头衔 + 一句话定位              │
├─────────────────────────────────────────┤
│ 履历时间轴（workExperience）              │
│  ├ 远算科技 · 前端 / PMO                  │
│  └ 融先科技 · 资管云                      │
├─────────────────────────────────────────┤
│ 能力矩阵（skillsDetail 分类摘要）          │
├─────────────────────────────────────────┤
│ 价值观 / 工作方式（3–5 条短句）            │
├─────────────────────────────────────────┤
│ 联系 CTA + 社交链接（真实 URL）            │
└─────────────────────────────────────────┘
```

**世界观命名**：顶栏 Tab 名 **「园主」**；页面标题可用「关于庭园」。

**导航**：Q5B — 顶栏独立 Tab，Logo 承担回首页职责。

**联系**：Q3A — 页内 **明文展示邮箱** + 社交链接；Footer 同步。

**数据**：复用 `src/data/site/personal.ts`，无需新表。

---

### 4.2 项目详情（/projects/[slug]）— **强烈推荐 P0**

**价值**：作品集站点标配；支撑分享、SEO、深度阅读。

**数据源（Q4A）**：扩展 Supabase **`all_projects`** 表，新增详情字段，不采用 MDX 静态案例。

**建议新增字段**：`slug`（唯一）、`role`、`period`、`tech_stack`（jsonb）、`highlights`（text[]）、`body`（markdown，可选）、`is_featured`（已有逻辑可复用）。

**最小展示**：title、description、tags、screenshots、links、tech_stack、period、role、highlights[]。

**交互**：上一/下一项目；相关技能跳转；助手「详细介绍这个项目」深链。

---

### 4.3 ~~博客 / 长文（/writing）~~ — **不独立立项（Q1B）**

**决策**：与 **草本集（/vibe）合并**，不新增 `/writing` 路由。

**实施要点**：

- vibe 表增加 `kind: 'note' | 'article'`（或等价字段）
- 长文支持 Markdown 正文；列表可按 kind 筛选或统一时间线
- 将 `personal.ts` 中 `blogPosts` 种子迁移入库或 vibe 种子数据
- RAG 索引 vibe 公开长文；来源链到 `/vibe/[slug]`
- 首页「最新输出」卡片取自 vibe 最近 article

---

### 4.4 联系页（/contact）— **降为 P2（Q3A）**

**决策**：邮箱在 **园主页 + 全站 Footer 明文展示**；联系转化主路径不再依赖独立页。

**可选补充**（P2）：轻量 `/contact` — 微信二维码、合作类型说明；助手 Contact Mode 保留为第二路径。

**价值**：Q3A 已覆盖 HR 最短路径（Footer / 园主 Tab → 邮箱）。

---

### 4.5 公开展柜（/showcase 或 /gallery）— **推荐 P2**

**定位（Q2A）**：`kb_records` 中 `is_public=true` 的**自动聚合只读视图**，非手工策展。

**场景**：技术片段、代码技巧——所有者逐条标记公开，展柜自动同步。

**与档案馆关系**：

- 档案馆 = 登录后完整 CRUD + 每条记录的公开开关
- 展柜 = `/knowledge/public` 或独立路由，仅渲染 `is_public` 记录

---

### 4.6 站点统计 / 开放数据（/stats）— **可选 P3**

**定位**：动态度量仪的完整版——项目数、技能分布、知识库条目、最近更新热力图。

**价值**：极客友好、透明感；需注意不暴露敏感数据。

---

### 4.7 变更日志 / 庭园志（/changelog）— **可选 P3**

**定位**：站点自身迭代记录（含 SpiritGarden 主题、RAG 上线等）。

**价值**：展示产品思维与持续维护；可手写 `content/changelog.md`。

---

### 4.8 内容工作台（Owner Dashboard）— **中长期 P2**

**背景**：子页 CRUD 分散在各自 Modal；知识库有 Cmd+K。

**方案**：登录后 `/studio` 统一入口：

- 待办：草稿、最近编辑
- 快捷：新建项目 / 笔记 / 知识 / 资源
- 状态：RAG 索引健康、broken links

**注意**：个人站不必做完整 CMS；**最小可用**即可。

## 5. 跨模块整合机会

### 5.1 内容 → RAG → 页面 闭环

```
录入（CRUD / Cmd+K）
    ↓ is_public 标记
索引（rag:index / 自动 reindex）
    ↓
助手回答 + 来源链接
    ↓
详情页（projects / vibe / knowledge 公开项）
```

**当前断点**：知识库 RLS 阻断索引；无详情页导致来源无法深链。

### 5.2 统一 Tag 体系

项目 tags、技能名、vibe tags、知识库 tags、资源 tags **可共享词表**（不必统一表结构，UI 层做「相关 content」推荐即可）。

### 5.3 叙事动线（访客旅程）

```
首页（5 秒建立认知）
  → 关于（信任）
  → 项目详情（深度）
  → 技能/资源（能力边界）
  → 助手（答疑）或 联系（转化）
```

建议在首页增加**引导式路径**（非强制教程）：「初次来访？从这里开始 →」。

## 9. 验收标准（立项后适用）

### 9.1 关于页（园主）

- [ ] 顶栏可见「园主」Tab；Logo 点击回 `/`（Q5B）
- [ ] 未登录访客 1 次点击到达 `/about`
- [ ] 履历、技能摘要与 `personal.ts` 一致
- [ ] 页内与 Footer **明文展示邮箱**（Q3A）；社交链接为真实 URL

### 9.2 项目详情

- [ ] 详情数据来自 Supabase `all_projects` 扩展字段（Q4A）
- [ ] 每个公开项目有可访问 `/projects/[slug]` URL
- [ ] 分享链接 OG 预览正常
- [ ] 助手回答可链到详情页

### 9.3 草本集长文（Q1B）

- [ ] vibe 支持 note / article 区分；长文可 Markdown 渲染
- [ ] 无独立 `/writing` 路由
- [ ] 原 `blogPosts` 内容可在 vibe 中阅读

### 9.4 公开内容策略（Q2A）

- [ ] `kb_records.is_public` 字段与 RLS 允许匿名读公开行
