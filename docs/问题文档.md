# 碎片知识库（Knowledge Base）问题审查文档

> 最后更新：2026-06-11。已修复项标注 ✅。

## 1. 严重功能异常或缺失 (Critical Issues)

- ✅ **CRUD**：已实现编辑（`EditRecordModal`）、删除（内联确认 + Server Action）
- ✅ **标签过滤**：`TagFilterBar` 使用 `Link` 跳转，过滤闭环可用
- ✅ **移动端录入**：`MobileFab` 触发 `open-quick-record` 事件

## 2. 性能与底层架构缺陷 (Architecture Issues)

- ✅ **标签体系**：`lib/knowledge/queries.ts` 分页扫描全库 tags（非 limit(100) 样本）
- ✅ **列表截断**：首屏 50 条 + `RecordList`「加载更多」
- ✅ **Signed URL 过期**：7 天 TTL + 图片 `onError` 重签

## 3. 防御性编程弱点 (Edge Cases)

- ✅ **文件体积**：粘贴与提交前均校验 5MB（`KB_MAX_FILE_SIZE`）
- ✅ **表单时序**：`QuickRecordModal` 移除 setTimeout 清表单，用 `isSubmitting` 锁

## 4. 全局 UX（2026-06-11 补充）

- ✅ **`alert()` / `window.confirm()`**：已替换为 Toast + `DeleteConfirmBar` 内联确认

## 仍待优化（非阻塞）

- tags 数据量极大时改 Supabase RPC 聚合
- `ProjectReader.tsx` 的 `<img>` 可换 `next/image`
